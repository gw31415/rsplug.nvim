# rsplug.nvim 0.4.0

Released 2026-07-24.

## Highlights

- Faster plugin installation and updates through parallel discovery, adaptive
  remote scheduling, snapshot catalog fast paths, and reuse of already-installed
  repositories.
- Faster GitHub downloads with `RSPLUG_GITHUB_TOKEN`, API revision resolution,
  codeload tarballs, connection pooling, and adaptive concurrency.
- More deterministic published pack directories: copy fallbacks no longer
  create hard-link aliases that can mutate an already-published snapshot.
- Improved lazy-loading and runtime behavior, including cleaner trigger state,
  corrected Lua module roots, and more reliable plugin/dependency loading.
- Clearer installer output with configuration summaries, fetch progress, build
  output, and actionable build errors.

## Release and validation

- All six workspace crates are published at version `0.4.0`.
- Release binaries are available for x86_64 and aarch64 Linux, x86_64 and
  aarch64 macOS, and x86_64 Windows.
- Every binary archive includes version verification and is accompanied by
  `SHA256SUMS` and build provenance attestation.
- The release workflow runs formatting, the full locked test suite, Clippy,
  package checks, and an immutable annotated-tag preflight before publication.

## Upgrade notes

The configuration format remains compatible with the previous release. Existing
lock files can be reused; run your normal update command when you want to refresh
locked revisions.

See the [full commit history](https://github.com/gw31415/rsplug.nvim/compare/v0.3.0...v0.4.0)
for implementation details.
